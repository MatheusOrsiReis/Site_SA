Fundamentos de Tecnologia de Redes


O que são Redes de Computadores?


Redes de computadores são sistemas que conectam dispositivos para que eles possam trocar informações. Esses dispositivos podem ser computadores, celulares, impressoras, servidores, smart TVs e até câmeras de segurança. Essas conexões podem ser feitas por cabos, Wi-Fi, fibra óptica ou até satélites. O objetivo principal é permitir que dados viajem entre equipamentos, como quando você abre um site, assiste a um vídeo, envia uma mensagem, compartilha um arquivo ou joga online. Sem redes de computadores, cada máquina seria isolada — não existiria internet, streaming, redes sociais, nuvem, nada.


O que é uma Rede mínima?


Uma rede mínima é a menor estrutura capaz de ser considerada uma rede. Ela é formada por dois dispositivos que conseguem se comunicar diretamente. Isso pode ser dois computadores ligados por um cabo, dois celulares conectados por Wi-Fi direto (hotspot) ou até um notebook conectado a um roteador. A partir dessa conexão mínima já existe troca de dados, endereços de rede e comunicação entre dispositivos.


O que é a Internet? O que compõe?


A internet é uma enorme rede global que interliga milhões de redes menores pelo mundo inteiro. Ela é composta principalmente por cabos de fibra óptica submarinos, servidores, data centers, provedores de serviços (ISP), roteadores de alta capacidade, satélites e sistemas que operam protocolos de comunicação, como DNS, IP, TCP e HTTPS. Quando você acessa um site ou assiste um vídeo, seus dados atravessam toda essa infraestrutura, passando por roteadores, cabos e servidores até chegar ao destino. A internet não é um único lugar, mas sim um conjunto gigantesco de estruturas distribuídas conectando tudo e todos.


O que é Web?


A web (World Wide Web) é um serviço que funciona dentro da internet. Enquanto a internet é a infraestrutura física e lógica, a web é o conjunto de páginas, sites e plataformas que acessamos pelo navegador. Quando você entra em sites como Google, YouTube, Mercado Livre ou redes sociais, está usando a web. Ela funciona com protocolos como HTTP e HTTPS, que definem como o navegador solicita páginas aos servidores e como esses servidores retornam textos, imagens, vídeos e códigos para serem exibidos.


O que é o mapa de fibra óptica?


O mapa de fibra óptica é uma representação visual de todos os cabos de fibra que transportam dados ao redor do mundo. Esses cabos são responsáveis por mais de 95% de toda a comunicação global. O mapa mostra rotas entre continentes, pontos de entrada dos cabos em países e a estrutura física que faz a internet funcionar. Ele é usado por provedores, empresas de tecnologia e governos para monitorar rotas, planejar expansões e prevenir falhas.






Componentes de hardware de uma rede Wi-Fi


Uma rede Wi-Fi é formada pelo modem, que recebe a internet do provedor, e pelo roteador Wi-Fi, que distribui essa conexão sem fios para os dispositivos. Esse roteador possui antenas internas ou externas que irradiam o sinal, além de uma placa de rádio responsável pela comunicação. Os dispositivos que se conectam ao Wi-Fi possuem placas de rede sem fio (como celulares, notebooks, TVs, videogames). Em ambientes maiores, podem existir repetidores, access points e controladores de rede para ampliar o alcance e manter a qualidade do sinal.


Componentes de hardware de uma rede local cabeada


Uma rede cabeada funciona com um modem conectado a um roteador, mas utiliza principalmente cabos Ethernet para transmitir os dados. Para distribuir a conexão para vários computadores, switches são usados. Cada cabo liga o switch aos computadores, que possuem placas de rede Ethernet integradas ou externas. Em empresas, essa estrutura costuma ficar organizada em racks com patch panels, guias de cabos, tomadas de rede e sistemas de gerenciamento, garantindo estabilidade e alta velocidade.


O que é um protocolo de comunicação?


Um protocolo de comunicação é um conjunto de regras que define como dispositivos trocam dados entre si. Ele determina o formato das mensagens, como elas são enviadas, como são recebidas, verificadas e como os dispositivos respondem. Sem protocolos, computadores não conseguiriam se entender. A internet funciona graças a vários protocolos, como TCP (transporte confiável), IP (endereçamento), DNS (resolução de nomes), HTTP (web), SMTP (e-mail) e muitos outros.


O que é o modelo OSI?


O modelo OSI é um padrão teórico dividido em sete camadas que explica como ocorre a comunicação de rede desde o sinal físico dos cabos até o funcionamento dos aplicativos. Cada camada tem uma função específica, como tratar erros, dividir pacotes, gerenciar conexões e entregar dados ao usuário. Embora não seja usado literalmente na prática, ele é essencial para estudar redes, porque separa a comunicação em etapas organizadas e fáceis de entender.


E o modelo TCP/IP?


O modelo TCP/IP é o modelo real utilizado na internet e é baseado em quatro camadas. Ele integra os principais protocolos de comunicação que fazem a internet funcionar, como o IP (endereçamento), o TCP (transporte confiável), o UDP (transporte rápido) e o HTTP (comunicação da web). Enquanto o OSI é uma referência teórica, o TCP/IP é a implementação prática usada diariamente em todo o mundo.


O que é IPv4 e IPv6?


O IPv4 é a versão antiga do protocolo IP que usa endereços de 32 bits, como 192.168.1.5. Ele já está ficando limitado, pois existem poucos endereços disponíveis no mundo. O IPv6 é a versão mais moderna e utiliza endereços de 128 bits, como 2804:14d:abcd::1, permitindo praticamente infinitos endereços. Além disso, o IPv6 traz melhorias em segurança, desempenho e organização da rede.



Como é configurado o IP e o TCP em uma programação backend com node.js?


Em Node.js, o programador normalmente apenas escolhe a porta e o endereço (host) onde o servidor vai rodar, como localhost ou 0.0.0.0. O Node usa o sistema operacional para cuidar do protocolo TCP/IP automaticamente. Ou seja, ao criar um servidor HTTP, o desenvolvedor só informa valores como app.listen(3000), e o sistema operacional gerencia conexões, pacotes TCP, IP, portas e envio de dados. O programador não precisa configurar o TCP “na mão”; tudo isso já é abstraído.


O que é o protocolo HTTP e HTTPS


O HTTP é o protocolo usado na web para comunicação entre navegadores e servidores. Ele funciona com requisições e respostas: o navegador pede uma página e o servidor devolve o conteúdo. Ele permite carregar textos, imagens, vídeos, formulários, scripts e APIs. É a base de praticamente todos os sites modernos.
Já o HTTPS é a versão segura do HTTP, adicionando criptografia com SSL/TLS. Ele protege os dados contra espionagem, roubo e adulteração, garantindo que a comunicação entre usuário e servidor esteja protegida. É o padrão atual de segurança da web, representado pelo cadeado na barra do navegador.


Exemplo de framework HTTP para Node.js


Um dos frameworks mais usados é o Express.js, que permite criar servidores, APIs e rotas de maneira rápida, simples e organizada. Ele facilita o uso do protocolo HTTP no ambiente Node.js e é padrão em grande parte dos projetos backend.




    const express = require('express')
    const app = express()
    const { SerialPort, ReadlineParser } = require('serialport')
    const readline = require('readline')
    
    const cors = require('cors')
    
    const PORT = 3000
    const hostname = 'localhost'
    
    const BAUD_RATE = 9600
    const SERIAL_PORT = "COM9"
    
    app.use(express.urlencoded({ extended: true }))
    app.use(express.json())
    app.use(cors())
    
    // Inicialização Arduino
    const arduino = new SerialPort({
        path: SERIAL_PORT,
        baudRate: BAUD_RATE,
    })
    
    const parserDados = arduino.pipe(new ReadlineParser({ delimiter: "\r\n" }))
    
    arduino.on("open", () => {
        console.log(`Conectado ao Arduino em ${SERIAL_PORT}`)
    })
    
    arduino.on("error", (err) => {
        console.error("Erro na comunicação serial", err)
    })
    
    // Parser das linhas recebidas
    parserDados.on("dados", (dad) => {
        const linha = dad.toString().trim()
    
        if (linha > 0) {
            console.log("Arduino:", linha)
        }
    })
    
    // Leitura do teclado
    const teclado = readline.createInterface({
        input: process.stdin,
        output: process.stdout,
    })
    
    teclado.on("line", (input) => {
        const comando2 = input.trim()
    
        if (comando2 === "s" || comando2 === "d") {
            arduino.write(comando2, (err) => {
                if (err) {
                    console.error("Erro ao enviar comando:", err)
                } else {
                    console.log(`Comando '${comando2}' enviado`)
                }
            })
        } else {
            console.log("Digite 's' (ligar) ou 'd' (desligar)")
        }
    })
    
    // Rotas HTTP
    app.get("/ligar", (req, res) => {
        arduino.write("s")
        res.status(200).json({ led: "ligado" })
    })
    
    app.get("/desligar", (req, res) => {
        arduino.write("d")
        res.status(200).json({ led: "desligado" })
    })
    
    // Servidor
    app.listen(PORT, hostname, () => {
        console.log(`Servidor rodando em ${hostname}:${PORT}`)
    })
